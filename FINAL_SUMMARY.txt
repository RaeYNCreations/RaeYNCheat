================================================================================
                         IMPLEMENTATION DETAILS
================================================================================

PASSKEY SYSTEM:
--------------
Format: "2003, December 15th:player-uuid"
Storage: Split into 5 parts (PART1="2003", PART2=", ", etc.)
Runtime: Reconstructed dynamically
Encoding: Base64 + string reversal
Obfuscation: ProGuard renames all methods/fields
Validation: Server compares client passkey with expected

VIOLATION TRACKING:
------------------
Two independent systems:
- checksumViolations (Map<UUID, Integer>)
- passkeyViolations (Map<UUID, Integer>)

Each has separate:
- Punishment configuration
- Escalation steps
- Admin commands

SENSITIVITY ANALYSIS:
--------------------
Detects false positives by analyzing file differences:
- NO_DIFFERENCE: 0 files changed
- LOW_DIFFERENCE: 1-2 files (might be testing)
- MEDIUM_DIFFERENCE: 3-9 files (suspicious)
- HIGH_DIFFERENCE: 10+ files (might be wrong modpack)
- TOTAL_MISMATCH: 80%+ different (completely wrong)

Smart punishment: Only punish MEDIUM, warn for others

CODE OBFUSCATION:
----------------
4 layers of protection:
1. Split key storage (5 parts)
2. Runtime reconstruction (StringBuilder)
3. Multi-encoding (Base64 + reversal)
4. ProGuard obfuscation (class/method/field renaming)

Result: Extremely difficult to reverse engineer

================================================================================
                         ADMIN COMMANDS
================================================================================

/raeynpunish <player>
  - Punish for checksum violations
  - Records checksum violation
  - Applies punishment based on violation count
  - Requires OP level 2

/raeynpasskeyban <player>
  - Punish for passkey violations
  - Records passkey violation
  - More aggressive punishment (passkey failures are suspicious)
  - Requires OP level 2

Both commands:
- Track violations independently
- Support progressive punishment
- Apply permanent bans with blacklist
- Can be configured separately
